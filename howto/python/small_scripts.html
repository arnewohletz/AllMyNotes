Run Python scripts from anywhereÔÉÅ

Python scripts (single *.py file programs) are especially handy, when those can be executed

  • from any current directory

  • without prior activating any virtual environment

but simply like in this example:

$ my_little_script.py

Important

These steps only work on macOS and Linux.

These steps require to have pyenv (including pyenv-virtualenv) installed on your system.

  1. Create a directory, where you like to save your scripts (e.g. ~/python_scripts).

  2. Add this directory to your PATH variable (add it to your shell‚Äôs profile, e.g. ~/.profile for bash or ~/.zprofile for zsh), e.g.:

    # Setting PATH to Python scripts directory
export PATH="/your/scripts/directory:$PATH"

  3. Open new Terminal window or source your shell initialization script. Example:

    $ source ~/.zprofile

  4. Create a virtual environment using pyenv for your scripts:

    $ pyenv virtualenv <PYTHON_VERSION> <VENV_NAME>

  5. Select your virtual environment as interpreter for the scripts directory:

    $ cd /your/scripts/directory
$ pyenv local <VENV_NAME>

  6. Create a new script file or copy your existing script(s) into your new scripts directory.

  7. Add this content on the very top of any script file in your scripts directory (replace existing shebang line, if applicable):

    #!/bin/zsh
"true" '''\'
pushd $(dirname "${0}") > /dev/null
"exec" "pyenv" "exec" "python" "$0" "$@"
'''

    Hint

    Depending on your operating system, you may have to alter the shell path (line 1) (e.g. /usr/bin/bash). Check your default shell by opening your Terminal and execute:

    $ echo $SHELL

You should now be able to execute any *.py script file within your scripts directory from anywhere in the shell, without the need to activate the script‚Äôs virtual environment first.
